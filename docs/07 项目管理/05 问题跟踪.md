## 3. 问题详情

### ISS-001: 本地AI模型对硬件要求较高

| 项目 | 内容 |
|------|------|
| 问题编号 | ISS-001 |
| 问题类型 | RISK |
| 优先级 | P1 |
| 发现日期 | 2026-02-28 |
| 发现人 | 赵后端 |
| 负责人 | 赵后端 |
| 状态 | 处理中 |

**问题描述**：
本地AI模型（如Qwen-7B）对硬件配置要求较高，需要至少16GB内存和支持GPU加速的显卡才能获得较好的响应速度。部分目标用户的办公电脑可能无法满足要求。

**影响分析**：
- 影响范围：私域环境AI功能
- 严重程度：可能导致部分用户无法使用本地AI功能
- 发生概率：中等

**应对措施**：
1. 提供多档模型配置（7B/3B/1B），根据硬件自动推荐
2. 优化模型加载和推理效率
3. 提供清晰的硬件配置说明文档
4. 考虑支持模型量化版本（GGUF 4-bit）

**进展记录**：

| 日期 | 进展内容 | 更新人 |
|------|----------|--------|
| 2026-02-28 | 问题识别并记录 | 赵后端 |

### ISS-002: PDF OCR功能依赖第三方库

| 项目 | 内容 |
|------|------|
| 问题编号 | ISS-002 |
| 问题类型 | RISK |
| 优先级 | P2 |
| 发现日期 | 2026-02-28 |
| 发现人 | 赵后端 |
| 负责人 | 赵后端 |
| 状态 | 已确认 |

**问题描述**：
PDF OCR功能需要依赖第三方OCR库（如Tesseract），可能存在：
1. 跨平台兼容性问题
2. 安装包体积增大
3. 识别准确率不稳定

**影响分析**：
- 影响范围：PDF导入功能
- 严重程度：中等，可文本PDF不受影响
- 发生概率：中等

**应对措施**：
1. 优先支持可选择文本的PDF直接提取
2. OCR功能作为可选组件，按需下载
3. 评估云端OCR服务作为公网环境备选方案

| 问题编号 | ISS-003 |

| 发现日期 | 2026-02-28 |
| 发现人 | 李产品 |
| 负责人 | 李产品 |
| 状态 | 处理中 |

**问题描述**：
《党政机关公文格式》(GB/T 9704-2012)中有些细节需要进一步确认：
1. 发文字号的具体格式变体
2. 签发人签名的位置规范
3. 附件说明的格式要求
4. 多页公文的页码格式

**应对措施**：
1. 查阅官方标准文件详细条款
2. 参考政府机关实际公文样本
3. 必要时咨询公文处理专家

### ISS-004: 向量数据库选型待确认

| 项目 | 内容 |
|------|------|
| 问题编号 | ISS-004 |
| 问题类型 | QA |
| 优先级 | P2 |
| 发现日期 | 2026-02-28 |
| 发现人 | 赵后端 |
| 负责人 | 赵后端 |
| 状态 | 新建 |

**问题描述**：
知识库功能需要本地向量数据库，目前有以下选项待评估：
1. LanceDB - Rust实现，与Tauri技术栈一致
2. Chroma - Python实现，社区活跃
3. SQLite + 向量扩展 - 轻量级方案

**待确认事项**：
- 性能对比（查询速度、内存占用）
- 与Tauri集成的复杂度
- 跨平台兼容性
- 许可证兼容性
