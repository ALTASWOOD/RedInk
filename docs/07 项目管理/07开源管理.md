## 2. 添加开源许可证

### 2.1 选择许可证

推荐使用 **MIT License**，原因：
- 简单宽松，便于商业和个人使用
- 允许修改、分发、私用和商用
- 只需保留版权声明

其他可选许可证：
| 许可证     | 特点                           | 适用场景           |
| ---------- | ------------------------------ | ------------------ |
| MIT        | 最宽松，几乎无限制             | 希望广泛传播       |
| Apache 2.0 | 提供专利授权，需保留声明       | 企业级项目         |
| GPL 3.0    | 强制开源衍生作品               | 希望保持开源       |
| BSD 3      | 类似MIT，禁止用名字做推广      | 学术项目           |

### 2.2 操作步骤

**方式一：在 GitHub 网页上添加（推荐）**

1. 打开仓库页面：https://github.com/ALTASWOOD/RedInk
2. 点击 **Add file** → **Create new file**
3. 文件名输入：`LICENSE`
4. 点击右侧 **Choose a license template**
5. 选择 **MIT License**
6. 填写年份和版权所有者名称
7. 点击 **Review and submit**
8. 点击 **Commit changes**

**方式二：本地创建后推送**

在项目根目录创建 `LICENSE` 文件，内容见下方模板。

## 4. 添加贡献指南

### 4.1 创建 CONTRIBUTING.md

在项目根目录创建 `CONTRIBUTING.md` 文件：

```markdown
# 贡献指南

感谢您对 RedInk 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告 Bug

1. 在 [Issues](https://github.com/ALTASWOOD/RedInk/issues) 中搜索是否已有相似问题
2. 如果没有，创建新的 Issue，请包含：
   - 问题描述
   - 复现步骤
   - 期望行为
   - 实际行为
   - 环境信息（操作系统、应用版本）
   - 相关截图或日志

### 提交功能建议

1. 在 Issues 中创建新的 Feature Request
2. 描述您期望的功能
3. 说明使用场景和预期效果

### 提交代码

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

## 开发环境

### 环境要求

- Node.js >= 18
- pnpm >= 8

### 本地开发

```bash
# 克隆仓库
git clone https://github.com/ALTASWOOD/RedInk.git
cd RedInk

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev
```

## 代码规范

- 使用 TypeScript 编写代码
- 遵循 ESLint 规则
- 提交前确保代码通过 lint 检查
- 提交信息遵循 [Conventional Commits](https://www.conventionalcommits.org/)

## Commit 信息规范

```
<type>(<scope>): <subject>

类型(type):
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 代码格式（不影响代码运行的变动）
- refactor: 重构
- test: 测试相关
- chore: 构建过程或辅助工具的变动
```

## 行为准则

请阅读我们的 [行为准则](CODE_OF_CONDUCT.md)。

## 联系我们

如有问题，欢迎在 Issues 中讨论。
```

labels: bug
assignees: ''
---

## Bug 描述
简明扼要地描述这个 bug。

1. 进入 '...'
2. 点击 '...'

描述实际发生了什么。

## 8. 创建首个 Release

### 8.1 操作步骤（GitHub 网页）

1. 打开仓库页面：https://github.com/ALTASWOOD/RedInk
2. 点击右侧 **Releases**
3. 点击 **Create a new release** 或 **Draft a new release**
4. 填写以下信息：

**Choose a tag:** 
- 点击输入框，输入 `v1.0.0`
- 点击 **Create new tag: v1.0.0 on publish**

**Release title:**
```
v1.0.0 - 首个正式版本
```

**Describe this release:**
```markdown
# RedInk v1.0.0

智能公文写作助手首个正式发布版本！

## 主要功能

### 🔐 公网/私域双模式
- 私域模式：数据完全本地处理，支持 Ollama 本地 AI
- 公网模式：使用云端 AI 服务，能力更强

### 📝 公文编写
- 支持多种公文类型模板（通知、报告、请示、批复等）
- 符合党政机关公文格式标准
- 富文本编辑器，支持表格、图片

### 🤖 AI 智能功能
- AI 生成：根据要点自动生成公文
- AI 审核：检查格式规范和内容质量
- AI 改写：润色、精简、扩展文本

### 📤 多格式导出
- 支持导出 DOCX、PDF、Markdown

## 系统要求

- Windows 10/11 (64位)
- macOS 10.15+
- 私域模式需安装 Ollama

## 安装说明

1. 下载对应系统的安装包
2. 运行安装程序
3. 按提示完成安装

## 私域模式配置

```bash
# 安装 Ollama
# 访问 https://ollama.com 下载安装

# 下载推荐模型
ollama pull qwen:7b
```

---

感谢使用 RedInk！如有问题请在 Issues 中反馈。
```

5. **Attach binaries:** 
   - 可以在本地打包后上传安装文件
   - 或者暂时跳过，后续添加

6. 选择 **Set as the latest release**

7. 点击 **Publish release**

---
